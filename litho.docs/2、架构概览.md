# System Architecture Documentation

## 1. Architecture Overview

### Architecture Design Philosophy

mns（Money Never Sleeps，Market Neutral Strategist）系统的设计哲学根植于"**配置驱动、数据隔离、业务解耦、本地优先**"四大原则，旨在为个人逆向投资者构建一个**无服务依赖、高纪律性、可复现**的自动化投资辅助工具。系统摒弃了传统金融平台的复杂后端架构，转而采用轻量级、单机运行的命令行模式，确保用户数据完全自主可控，规避云端隐私泄露与服务中断风险。

其核心思想是：**将投资策略从代码中剥离，固化为可读、可改、可审计的配置文件；将数据持久化与业务逻辑分离，确保状态可回溯；将外部依赖封装为可替换的适配层，提升系统健壮性**。这种设计不仅契合个人投资者对"纪律性交易"的心理需求，也完美适配Rust语言在内存安全、零成本抽象和编译期验证方面的优势，使系统兼具高性能与高可靠性。

### Core Architecture Patterns

mns系统采用**分层架构** 作为主干，并融合以下经典设计模式：

| 模式 | 应用场景 | 价值体现 |
|------|----------|----------|
| **分层架构** | 划分工具支持层、基础设施层、核心业务层 | 实现关注点分离，降低模块耦合，提升可维护性 |
| **依赖倒置原则（DIP）** | 所有模块依赖抽象接口（数据模型、函数签名），而非具体实现 | 便于单元测试（如mock数据库或API） |
| **外观模式** | `cli.rs` 作为统一入口，隐藏内部模块调用复杂性 | 提供简洁、一致的用户交互界面 |
| **适配器模式** | `sentiment.rs` 封装 CNN API，返回统一结构体 | 隔离外部服务变化，降低上游API变更影响 |
| **事务性模式** | 数据库操作通过原子事务完成（`db.rs`） | 保障金融数据一致性，避免账实不符 |
| **策略模式** | 策略引擎内部通过配置动态决定买入/卖出权重 | 实现策略参数化，无需修改代码即可调整投资逻辑 |

### Technology Stack Overview

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| **编程语言** | Rust 1.92+ (edition 2024) | 内存安全、零成本抽象、编译期错误检查、无GC延迟，适合金融级稳定系统 |
| **数据获取** | reqwest + CNN API | 直接调用 CNN Fear & Greed API（股票市场恐贪指数） |
| **配置管理** | TOML + `serde` | 人类可读、结构清晰、支持嵌套、与Rust结构体无缝序列化 |
| **数据持久化** | SQLite 3.40+ (`rusqlite`) | 单文件、零配置、ACID事务、无服务依赖、跨平台 |
| **HTTP客户端** | `reqwest` + `tokio` | 异步非阻塞、支持HTTPS、自动重试、JSON反序列化集成良好 |
| **CLI解析** | `clap` (v4) | 功能强大、类型安全、自动生成帮助文档、支持子命令嵌套 |
| **表格输出** | `comfy-table` | 美观的ASCII表格渲染，支持样式和颜色 |
| **构建工具** | Cargo | 标准Rust包管理与构建系统，支持跨平台编译与依赖管理 |

> ✅ **技术选型总结**：系统采用"**最小可行技术栈**"策略，所有组件均为成熟、稳定、无运行时依赖的开源库，确保系统可编译为单个二进制文件（`mns`），部署零环境依赖，符合"个人投资助手"的轻量定位。

## 2. System Context

### System Positioning and Value

mns是一个面向**个人逆向投资者**的本地命令行金融决策辅助工具，其核心价值在于**将行为金融学中的"恐惧与贪婪"理论转化为可执行、可复盘、可自动化**的投资纪律系统。

在传统投资场景中，投资者常因情绪波动（如恐慌性抛售、贪婪性追高）导致非理性决策。mns通过以下方式提供差异化价值：

- **自动化逆向策略**：当市场情绪极度恐惧（<20）时，系统自动推荐加仓（80%）；当极度贪婪（>75）时，推荐获利了结（60%）。
- **防止"接飞刀"**：基于配置阈值（如亏损≥30%）自动过滤高风险资产，避免在下跌趋势中盲目抄底。
- **动态风险对冲**：根据情绪等级动态调整买入/卖出比例，实现"情绪对冲"。
- **策略回测验证**：基于历史数据验证策略有效性，找到最优配置参数。
- **每日复盘报告**：生成结构化中文日报，强制用户进行"非情绪化"决策回顾，提升投资纪律。

该系统不是交易平台，也不是投资顾问，而是**用户投资行为的"纪律教练"** —— 通过数据与规则，将主观判断转化为客观行动指南。

### Storage Design

系统采用**本地文件系统 + SQLite** 的双层持久化架构：

#### 1. 配置存储（TOML）
- **路径**：`~/.mns/config.toml`
- **默认配置（保守配置）**：
  ```toml
  [settings]
  annualized_target_low = 10.0
  annualized_target_high = 15.0
  min_holding_days = 45
  min_absolute_profit_days = 120
  max_contrarian_weight = 2.0

  [allocation]
  us_stocks = 55.0
  cn_stocks = 25.0
  counter_cyclical = 20.0

  [thresholds]
  extreme_fear = 30.0
  fear = 45.0
  neutral = 55.0
  greed = 70.0

  [buy_ratio]
  extreme_fear = 60.0
  fear = 35.0
  neutral = 0.0
  greed = 0.0

  [sell_ratio]
  extreme_greed_target_high = 50.0
  greed_target_high = 40.0
  ```

#### 2. 数据存储（SQLite）
- **路径**：`~/.mns/mns.db`
- **表结构**：cash, positions, transactions, fear_greed_snapshots

## 5. Data Flow

### Report Generation Flow

```
cmd_report()
  1. sentiment::fetch_fear_greed_index()   → score (0-100)
  2. db::Database::open()                  → cash balance, positions
  3. db::save_fear_greed_snapshot()        → persist daily snapshot
  4. strategy::calculate_sell()           → Vec<SellSuggestion>
  5. strategy::calculate_buy(sell_refs)    → BuySuggestion
  6. strategy::check_risk_warnings()       → Vec<RiskWarning>
  7. report::generate_report()             → formatted text
  8. report::save_report()                 → write to reports/
```

### Backtest Flow

```
cmd_backtest()
  1. load embedded CSV data (monthly_real_final.csv)
  2. parse multi-asset data (nasdaq, dividend_low_vol, gold_cny)
  3. run_multi_asset_backtest() for 美股+红利低波+黄金
  4. run_buy_and_hold() for baseline
  5. run_backtest() with multiple config variants
  6. print_comparison() showing all results
```

### Price Update Flow

```
cmd_update_prices()
  1. db.list_positions() → get all positions
  2. for each position:
     - if 6-digit code: fetch_from_tiantian(code)
     - if letter code: fetch_from_yahoo(code)
  3. db.update_position_price(code, price)
  4. display update results table
```

## 6. Key Design Patterns

| 模式 | 说明 |
|------|------|
| **加权平均成本** | `buy_position()` 重算成本价 `(old×cost + new×price) / (old+new)` |
| **年化收益（含最小天数）** | `(current/cost)^(365/days) - 1`，不足 `min_holding_days` 时显示 N/A |
| **绝对收益** | `(current - cost) / cost`，用于长期止盈 |
| **逆向买入分配** | 权重 `max(1.0, cost/current)`，浮亏越多权重越高 |
| **情绪区间** | 5 区间：极度恐慌(<20)、恐慌(20-40)、中性(40-55)、贪婪(55-75)、极度贪婪(>75) |
| **卖出矩阵** | 区间×收益水平 的二维决策表 |
| **双重卖出触发** | 年化收益（主）+ 绝对收益≥30%（辅）|
| **事务性 DB** | SQLite 事务保障金融数据一致性 |
| **嵌入历史数据** | 通过 `include_str!` 将 CSV 编入二进制 |
