# 策略回测引擎

## 1. 模块概述

### 模块定位

`backtest.rs` 是 MNS 系统的核心业务模块，负责**策略有效性验证**。该模块是代码量最大的模块（约1500行），实现了完整的历史数据回测框架，支持单资产和多资产组合的回测分析。

### 核心价值

| 价值点 | 说明 |
|--------|------|
| **策略验证** | 使用2016-2025年真实历史数据验证逆向策略有效性 |
| **参数优化** | 支持多组配置对比，找到最优策略参数 |
| **风险量化** | 计算最大回撤、年化收益等关键指标 |
| **基准对比** | 与"买入持有"策略对比，证明策略超额收益 |
| **多资产支持** | 支持美股+红利低波+黄金的多资产组合回测 |

## 3. 核心 API

### run_backtest - 单资产回测

```rust
pub fn run_backtest(config: &AppConfig, bt_config: &BacktestConfig) -> BacktestResult
```

**功能**：对单一资产（如标普500 ETF）执行逆向策略回测。

**处理流程**：
```
1. 加载嵌入的历史数据（FGI + SP500）
2. 聚合为月度数据
3. 按时间顺序遍历每个月：
   - 年度注资（每年3月末）
   - 判断情绪区间变化
   - 调用 strategy::calculate_sell_suggestions()
   - 调用 strategy::calculate_buy_suggestions()
   - 执行交易并更新状态
   - 记录月度快照
4. 计算最终收益指标
5. 返回 BacktestResult
```

**交易触发条件**：
| 条件 | 说明 |
|------|------|
| 区间变化 | 情绪从一区间进入另一区间时触发 |
| 新资金注入 | 有新资金且处于可买入区间（恐慌及以下） |
| 冷却期后 | 距上次交易≥3个月且处于可买入区间 |

## 5. 关键算法

### 年化收益率计算

```rust
let days = (end_date - start_date).num_days() as f64;
let years = days / 365.0;
let annualized_return = (final_value / total_inflow).powf(1.0 / years) - 1.0;
```

**公式**：$r = (V_f / V_i)^{1/n} - 1$

其中：
- $V_f$ = 期末总资产
- $V_i$ = 总投入本金
- $n$ = 持有年数

---

### 最大回撤计算

```rust
let mut max_value: f64 = 0.0;
let mut max_drawdown: f64 = 0.0;
for mv in &monthly_values {
    max_value = max_value.max(mv.total_value);
    let drawdown = (max_value - mv.total_value) / max_value;
    max_drawdown = max_drawdown.max(drawdown);
}
```

**逻辑**：遍历历史资产曲线，记录从峰值回落的最大幅度。

---

### 情绪区间判定

```rust
fn get_zone_name(score: f64, config: &AppConfig) -> &'static str {
    if score < config.thresholds.extreme_fear {
        "极度恐慌"
    } else if score < config.thresholds.fear {
        "恐慌"
    } else if score < config.thresholds.neutral {
        "中性"
    } else if score < config.thresholds.greed {
        "贪婪"
    } else {
        "极度贪婪"
    }
}
```

**阈值来源**：`config.thresholds`（默认：极度恐慌<30, 恐慌<45, 中性<55, 贪婪<70）。

---

### 加权平均成本

```rust
// 买入时更新成本价
let total_shares = state.position.shares + buy_shares;
state.position.cost_price = 
    (state.position.shares * state.position.cost_price + buy_shares * price) / total_shares;
```

**公式**：$新成本 = \frac{旧份额 \times 旧成本 + 新份额 \times 新价格}{总份额}$

## 9. 设计决策

### 为什么使用嵌入式数据？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **嵌入式CSV** | 单二进制部署、无外部依赖、数据版本固定 | 更新数据需重新编译 |
| 外部文件 | 灵活更新 | 部署复杂、路径问题 |
| 数据库 | 查询灵活 | 引入额外依赖、过度设计 |

**结论**：对于个人工具，嵌入式方案最优。

---

### 为什么按月回测而非按日？

1. **FGI数据源限制**：CNN API提供的历史数据通常以月度聚合更稳定
2. **交易频率匹配**：逆向策略本质是低频交易，月度精度足够
3. **计算效率**：月度数据量小，回测速度快

---

### 为什么多资产使用固定比例？

```rust
let us_ratio = config.allocation.us_stocks / 100.0;
let cn_ratio = config.allocation.cn_stocks / 100.0;
let gold_ratio = config.allocation.counter_cyclical / 100.0;
```

**原因**：
1. 简化策略逻辑，避免引入动态资产配置的复杂性
2. 用户可在配置文件中调整比例
3. 保持策略的可解释性和可复现性

## 10. 限制与改进方向

### 当前限制

| 限制 | 说明 |
|------|------|
| **单一数据源** | 仅使用CNN FGI，缺少其他情绪指标 |
| **固定时间窗口** | 回测期间固定为2016-2025，无法自定义 |
| **无交易成本** | 未考虑滑点、佣金等交易成本 |
| **无分红复投** | 未处理分红再投资场景 |

### 改进方向

| 改进项 | 优先级 | 说明 |
|--------|--------|------|
| 自定义回测区间 | 高 | 支持 CLI 参数指定起止日期 |
| 交易成本模拟 | 中 | 添加佣金、滑点配置项 |
| 多情绪指标 | 中 | 支持 VIX、Put/Call Ratio 等 |
| 分红处理 | 低 | 添加分红再投资逻辑 |
| 可视化输出 | 低 | 生成资产曲线图 |
