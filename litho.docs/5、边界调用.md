# System Boundary Interface Documentation

This document describes the system's external invocation interfaces, including CLI commands, API endpoints, configuration parameters, and other boundary mechanisms.

## Command Line Interface (CLI)

### init

**Description**: 初始化应用配置文件、数据库和报告目录。

**Source File**: `src/main.rs`

**Usage Examples**:

```bash
mns init
```

### config

**Description**: 查看或修改配置项，支持动态查询和设置。

**Source File**: `src/main.rs`

**Arguments**:

- `key` (string): optional - 配置项的点路径键名，如 'settings.annualized_target_low'
- `value` (string): optional - 要设置的配置值，仅在设置时使用

**Usage Examples**:

```bash
mns config
```

```bash
mns config annualized_target_low
```

```bash
mns config annualized_target_low 12.0
```

### cash

**Description**: 查看当前现金余额。

**Source File**: `src/main.rs`

**Usage Examples**:

```bash
mns cash
```

### cash set

**Description**: 设置现金余额为指定值（不能为负）。

**Source File**: `src/main.rs`

**Arguments**:

- `amount` (float): required - 要设置的现金金额（单位：人民币）

**Usage Examples**:

```bash
mns cash set 10000
```

### cash add

**Description**: 向现金余额增加指定金额。

**Source File**: `src/main.rs`

**Arguments**:

- `amount` (float): required - 要增加的现金金额（单位：人民币）

**Usage Examples**:

```bash
mns cash add 500
```

### portfolio

**Description**: 显示当前资产持仓列表，包括成本价、现价、市值、年化收益等。

**Source File**: `src/main.rs`

**Usage Examples**:

```bash
mns portfolio
```

### add

**Description**: 添加新资产到投资组合，需指定代码、名称和类别。

**Source File**: `src/main.rs`

**Arguments**:

- `code` (string): required - 资产代码（如 AAPL、600519）
- `name` (string): required - 资产名称（如 苹果）
- `category` (string): required - 资产类别，可选：us_stocks、cn_stocks、counter_cyclical

**Usage Examples**:

```bash
mns add AAPL 苹果 us_stocks
```

### buy

**Description**: 买入指定资产的份额。

**Source File**: `src/main.rs`

**Arguments**:

- `code` (string): required - 资产代码
- `shares` (float): required - 买入份额
- `price` (float): required - 买入价格（单位：人民币）

**Usage Examples**:

```bash
mns buy AAPL 10.5 150.2
```

### sell

**Description**: 卖出指定资产的份额。

**Source File**: `src/main.rs`

**Arguments**:

- `code` (string): required - 资产代码
- `shares` (float): required - 卖出份额
- `price` (float): required - 卖出价格（单位：人民币）

**Usage Examples**:

```bash
mns sell AAPL 5.0 160.0
```

### price

**Description**: 查询或更新资产的当前价格。

**Source File**: `src/main.rs`

**Arguments**:

- `code` (string): required - 资产代码
- `price` (float): optional - 要更新的价格，不提供时仅查询

**Usage Examples**:

```bash

```

```bash

```

### sentiment

**Description**: 获取并显示当前CNN恐惧与贪婪指数，并保存至数据库。

**Source File**: `src/main.rs`

**Usage Examples**:

```bash
mns sentiment
```

### report

**Description**: 生成综合投资策略报告，包含情绪分析、买卖建议、风险预警，并保存为文件。

**Source File**: `src/main.rs`

**Usage Examples**:

```bash
mns report
```

### history

**Description**: 显示最近的交易历史记录，支持限制条数。

**Source File**: `src/main.rs`

**Arguments**:

- `limit` (integer): optional - 显示交易记录的最大条数，默认为全部

**Usage Examples**:

```bash
mns history
```

```bash

```

## Integration Suggestions

### 自动化定时任务

建议通过系统定时任务（如cron）定期执行 `mns sentiment` 和 `mns report` 命令，实现每日自动更新情绪数据和生成策略报告。

**Example Code**:

```
0 9 * * * /path/to/mns sentiment && /path/to/mns report
```

**Best Practices**:

- 将报告输出目录挂载至云存储或邮件通知系统
- 确保配置文件中 report_output_dir 指向可访问路径
- 使用日志记录定时任务执行状态

### 配置管理集成

建议将 config.toml 配置文件纳入版本控制系统（如Git），实现策略参数的可追溯与多环境部署（如测试/生产）。

**Example Code**:

```
git add .mns/config.toml && git commit -m "update target return to 12%"
```

**Best Practices**:

- 为不同用户或策略模板创建配置文件变体
- 使用环境变量覆盖敏感配置（如API URL）
- 在CI/CD流程中验证配置文件有效性

### 外部数据源增强

建议扩展支持多源市场情绪数据（如A股投资者情绪指数），通过配置动态切换数据源，提升策略的本地适应性。

**Example Code**:

```
mns config api.fear_greed_url "https://alternative-sentiment-api.com/data"
```

**Best Practices**:

- 在 AppConfig 中增加 source_type 字段用于区分数据源
- 为每个数据源实现独立的 fetch 模块
- 增加配置校验确保新源返回兼容格式
