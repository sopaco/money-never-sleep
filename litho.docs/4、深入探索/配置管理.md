# 配置管理模块技术文档

## 概述

**配置管理模块**是 mns 系统的基础设施核心，负责加载、验证、管理和动态调整系统运行参数。系统采用 **TOML 格式配置文件** 作为唯一真相来源，所有策略参数、风险阈值、资产分配比例均通过配置注入，实现"**策略与代码分离，规则与执行解耦**"的设计目标。

本模块提供：
- 配置文件加载与验证
- 运行时动态读写（`mns config` 命令）
- 情绪区间映射
- 买入/卖出比例决策矩阵

---

## 模块职责与定位

| 维度 | 说明 |
|------|------|
| **所属领域** | 基础设施层（Infrastructure Domain） |
| **核心职责** | 管理系统运行参数，提供策略决策所需的规则数据 |
| **输入** | `~/.mns/config.toml` 文件 |
| **输出** | `AppConfig` 结构体，供策略引擎、报告模块等使用 |
| **依赖项** | `serde` (序列化)、`toml` (解析)、`dirs` (路径)、`anyhow` (错误处理) |
| **被依赖方** | 所有业务模块（strategy.rs、report.rs、main.rs） |

## 配置结构详解

### 完整配置结构

```rust
pub struct AppConfig {
    pub settings: Settings,      // 全局策略参数
    pub allocation: Allocation,  // 资产分配比例
    pub thresholds: Thresholds,  // 情绪区间阈值
    pub buy_ratio: BuyRatio,     // 各区间买入比例
    pub sell_ratio: SellRatio,  // 卖出决策矩阵
    pub api: ApiConfig,         // API 配置（保留字段）
}

pub struct Settings {
    pub annualized_target_low: f64,      // 年化止盈低线（开始减仓）
    pub annualized_target_high: f64,    // 年化止盈高线（大笔减仓）
    pub min_holding_days: i32,          // 最小持仓天数
    pub min_absolute_profit_days: i32,  // 绝对收益最小天数
    pub max_contrarian_weight: f64,    // 逆向权重上限
    pub report_output_dir: String,      // 报告输出目录
}

pub struct Allocation {
    pub us_stocks: f64,         // 美股占比
    pub cn_stocks: f64,        // A股占比
    pub counter_cyclical: f64, // 逆周期资产占比
}

pub struct Thresholds {
    pub extreme_fear: f64,     // 极度恐慌阈值
    pub fear: f64,             // 恐慌阈值
    pub neutral: f64,          // 中性阈值
    pub greed: f64,            // 贪婪阈值
}

pub struct BuyRatio {
    pub extreme_fear: f64,     // 极度恐慌买入比例
    pub fear: f64,             // 恐慌买入比例
    pub neutral: f64,          // 中性买入比例
    pub greed: f64,            // 贪婪买入比例
}

pub struct SellRatio {
    pub extreme_greed_target_high: f64,    // 极度贪婪高收益减仓
    pub extreme_greed_target_low: f64,     // 极度贪婪低收益减仓
    pub extreme_greed_below_target: f64,   // 极度贪婪未达标减仓
    pub greed_target_high: f64,            // 贪婪高收益减仓
    pub greed_target_low: f64,             // 贪婪低收益减仓
    pub neutral_target_high: f64,          // 中性高收益减仓
}
```

## 核心业务逻辑

### 1. 情绪区间映射

```rust
pub fn sentiment_zone(&self, score: f64) -> &'static str {
    if score < self.thresholds.extreme_fear {
        "极度恐慌"
    } else if score < self.thresholds.fear {
        "恐慌"
    } else if score < self.thresholds.neutral {
        "中性"
    } else if score < self.thresholds.greed {
        "贪婪"
    } else {
        "极度贪婪"
    }
}
```

### 2. 买入比例获取

```rust
pub fn buy_ratio_for(&self, score: f64) -> f64 {
    if score < self.thresholds.extreme_fear {
        self.buy_ratio.extreme_fear
    } else if score < self.thresholds.fear {
        self.buy_ratio.fear
    } else if score < self.thresholds.neutral {
        self.buy_ratio.neutral
    } else {
        self.buy_ratio.greed
    }
}
```

### 3. 卖出比例获取（二维矩阵）

```rust
pub fn sell_ratio_for(&self, score: f64, annualized_pct: f64) -> f64 {
    // 恐慌区间不卖
    if score < self.thresholds.neutral {
        return 0.0;
    }
    
    // 判断收益水平
    let target_low = self.settings.annualized_target_low;
    let target_high = self.settings.annualized_target_high;
    
    // 根据情绪和收益查表
    if score >= self.thresholds.greed {
        // 极度贪婪
        if annualized_pct >= target_high {
            self.sell_ratio.extreme_greed_target_high
        } else if annualized_pct >= target_low {
            self.sell_ratio.extreme_greed_target_low
        } else {
            self.sell_ratio.extreme_greed_below_target
        }
    } else {
        // 贪婪区间
        if annualized_pct >= target_high {
            self.sell_ratio.greed_target_high
        } else if annualized_pct >= target_low {
            self.sell_ratio.greed_target_low
        } else {
            0.0  // 未达标不卖
        }
    }
}
```

## 配置验证

系统启动时自动验证配置合法性：

```rust
pub fn validate(&self) -> Result<()> {
    // 1. 资产分配之和必须为100%
    let alloc_sum = self.allocation.us_stocks 
        + self.allocation.cn_stocks 
        + self.allocation.counter_cyclical;
    if (alloc_sum - 100.0).abs() > 0.01 {
        bail!("资产配置比例之和必须为 100%，当前: {}%", alloc_sum);
    }
    
    // 2. 持仓天数不能为负
    if self.settings.min_holding_days < 0 {
        bail!("最小持仓天数不能为负数: {}", self.settings.min_holding_days);
    }
    
    // 3. 买入比例不能为负
    if self.buy_ratio.extreme_fear < 0.0 || self.buy_ratio.fear < 0.0 {
        bail!("买入比例不能为负数");
    }
    
    Ok(())
}
```

## 动态配置管理

### 查看配置

```bash
mns config                          # 查看所有配置
mns config settings.annualized_target_low  # 查看单个配置项
```

### 修改配置

```bash
mns config settings.annualized_target_low 15.0
mns config allocation.us_stocks 70.0
```

配置修改后立即生效，无需重启。

## 配置驱动设计优势

| 优势 | 说明 |
|------|------|
| **策略可调整** | 无需修改代码即可调整投资策略 |
| **环境隔离** | 开发/测试/生产环境可使用不同配置 |
| **版本可控** | 配置文件可纳入 Git 管理，追踪变更历史 |
| **用户友好** | TOML 格式易读易编辑，非程序员也能理解 |
| **实时生效** | 运行时动态修改，无需重新编译 |

## 总结

**配置管理模块**是 mns 系统的"大脑"，通过将投资策略参数化，实现了：

1. **策略与代码分离** — 参数变更无需重新编译
2. **规则透明可审计** — 所有决策逻辑可追溯
3. **个性化适配** — 用户可根据风险偏好调整参数
4. **历史验证** — 配置参数已通过回测验证有效性
